Economy Simulator

Fixed 72-item spinner · prototype balancing tool

Run

Starting resources

Player skill

Stage modes use a success/fail route: a successful attempt awards the HIGH tier, a failed attempt the LOW tier. Player type sets the success chance.

Survey strikes (shields)

Survey Says & Keep or Pass use a three-strikes rule (fail on the 3rd). Shields auto-absorb strikes (1 each), so a weak player can burn up to 3 shields per round to avoid failing; a strong player rarely strikes. Strike count (0–3) is a weighted draw by player type — enter 4 weights for [0, 1, 2, 3] strikes.

Energy

Time-based regen per the energy spec. The cap limits only passive regen — rewards and events can push energy above it.

Daily schedule

People sleep. When enabled, the player only plays during a waking window each day; energy still regenerates (capped) overnight and attacks still occur 24/7 — only play is confined to waking hours. The "away when empty" gap above still applies during waking hours; if it runs into the night, the player sleeps until morning.

Jackpot pool

The jackpot mirrors coin gains: whenever the player earns coins, the same amount (× rate) is added to the pool. Collected when Fast Money triggers.

Fast Money

Survey points fill the progress bar (targets from the Fast Money CSV). Reaching a target lets the player collect the whole jackpot pool. Jackpot fills separately via coin collection.

Attacks / damage

Attacks are time-based: the player is attacked a fixed number of times per simulated day, spaced by elapsed time (so attacks also land while away regenerating energy).

Shields

Repair

When on, repairs are allowed; the active spender persona decides when to repair vs. upgrade and how aggressively.

Spending / builder

The spender persona models how the player spends coins — distinct economy-pressure profiles for comparison.

Persona parameters

Reward values

Per-outcome parameters. Grouped by spinner category. Edit freely — these drive all payouts.

Source data (JSON)

Override the raw datasets. Paste or upload JSON, then Apply. Changes take effect on the next run. Reset restores the values baked from the source CSVs.

Spinner outcomes

Array of { index, category, outcome }. The outcome name links to its reward family and the category drives chart grouping.

Fast Money target values

Array of { queue, target }, in trigger order. Each target is the survey-point goal for that Fast Money trigger.

Upgrade (build) costs

Object keyed level → building → [star1, star2, star3] coin costs to build/upgrade.

Repair costs

Object keyed level → building → [star1, star2, star3] coin costs to repair after an attack.

Configure on the left, then press Run Simulation. Defaults are placeholders — tune reward values and policies to balance the economy.
